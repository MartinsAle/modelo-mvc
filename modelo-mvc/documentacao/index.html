O arquivo index.php

É nesse arquivo que serão montadas todas as páginas do site. Ele possui uma lógica de inclusão dos arquivos que estão na pasta views.

A estrutura de montagem das páginas está representada no exemplo a seguir. Sendo que este deve ser montado seguindo a lógica de cada template que é utilizado para criar o site. Lembrando que o código PHP principal deve ser mantido inalterado para que não ocorra nenhum problema na inclusão dos arquivos de visualização. (O código abaixo segue o exemplo do site teste encontrato nesse link)

O arquivo 404.php

Esse arquivo contém o HTML da página de erro quando uma página que o usuário procura não é encontrada.

O arquivo .htaccess

Nesse arquivo colocamos redirecionamentos via HTTP, onde habilitamos a reescrita de URL, forçamos redirecionamento para HTTPS, adicionamos compressão GZIP, etc.

A pasta control

Esta pasta contém todos os arquivos PHP de controle da aplicação.

A pasta model

Dentro desta pasta existem mais duas pastas: a pasta Bean e a pasta DAO. Dentro da pasta Bean se encontram os arquivos PHP que contém as classes que serão utilizadas na aplicação.
Dentro da pasta DAO ficam todos os arquivos PHP que controlam a lógica da aplicação.

A pasta views

Nesta pasta se encontram todos os arquivos de visualização das páginas. Além disso, é aqui que colocamos os arquivos css, js e imagens. Esses três últimos citados se encontram dentro da pasta assets.
Aqui se econtra outro arquivo index.php, mas esse, por sua vez, representa a visualização da página principal do site (a home).
Aqui existe também a pasta include. Dentro dela temos os arquivos de scripts, do head, do menu, do footer (rodapé), o arquivo config.php e o arquivo controle_codigos_php_paginas.php que carrega os códigos de manipulação de dados PHP das demais páginas.

O arquivo scripts contém a chamada de todos os scripts (arquivos .js) que serão utilizados nas páginas.

O arquivo head contém a chamada dos arquivos .css e outras tags de definição das páginas HTML como a tag description, author, etc.

O arquivo menu contém o HTML do menu principal do site.

O arquivo footer contém o HTML do rodapé do site.

O arquivo config.php contém o código que define o caminho da pasta onde estão os arquivos HTML de visualização e também cria a lógica de definição e captura das URL.

O arquivo controle_codigos_php_paginas.php contém a inclusão dos arquivos de manipulação PHP de cada página de acordo com a URL. Exemplo: na página contato.php (URL = contato) existe um fomulário onde os dados precisam ser capturados. Dentro do arquivo de controle de códigos PHP se captura a URL contato e se inclui o arquivo que possui a lógica para capturar os dados do formulário da página.

Há também duas pastas: a codigos-php-paginas e a do phpmailer.

Dentro da pasta codigos-php-paginas estão os arquivos de manipulação PHP de cada página HTML. Estes contém os códigos que, por exemplo, capturam os dados ou executam um determinado comando de uma certa página.

Dentro da pasta phpmailer estão os arquivos PHP para enviar e-mail.
Contém os arquivos com as classes dos protocolos de envio de e-mail, um arquivo de autoload que chama o arquivo do protocolo utilizado para enviar e-mail e um arquivo de exemplo de envio de e-mail.

A pasta library

A pasta library contém o CRUD da aplicação e a configuração do banco de dados a ser utilizado.

Aqui encotramos duas pastas: a pasta config e a pasta data.

Dentro da pasta config está o arquivo config.php. Nele estão as constantes de configuração do banco de dados a ser utilizado.

Dentro da pasta data está o arquivo AcessoDados.php e o arquivo DataBase.php. No arquivo AcessoDados.php temos a criação do CRUD com as funções de CREATE, READ, UPDATE e DELETE e da conexão com o banco de dados via PDO.
No arquivo DataBase.php temos a criação do objeto db (data base) que será utilizado nos métodos dos arquivos da pasta DAO para inserir, atualizar, ler e exlcuir dados.
